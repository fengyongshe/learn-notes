HControl HA安装与配置文档

作者：姚磊

本文用来描叙实现HControl HA具体步骤。假设已经成功申请了虚拟IP(10.133.17.241)，主从节点分别是bdi_A(10.133.17.5)、bdi_B(10.133.17.9)。
安装时请注意在HControl Server(主/从)节点上除了Server本身、HControl Agent和Client,最好不要安装HControl的其他服务组件。

1 配置/etc/hosts(各节点)

vi /etc/hosts 加入以下配置

10.133.17.241   bdi_hcontrol_server
10.133.17.5     bdi_A
10.133.17.9     bdi_B
#其它agent配置

2 安装配置PostgreSQL 9.3(主/从)

yum install http://yum.postgresql.org/9.3/redhat/rhel-6-x86_64/pgdg-redhat93-9.3-1.noarch.rpm
yum install postgresql93-server postgresql93-contrib
service postgresql-9.3 initdb

vi /etc/profile

export PGDATA=/var/lib/pgsql/9.3/data
export PATH=$PATH:/usr/pgsql-9.3/bin

vi /var/lib/pgsql/9.3/data/postgresql.conf

listen_addresses = '*'
wal_level = hot_standby
max_wal_senders = 3
hot_standby = on

vi /var/lib/pgsql/9.3/data/pg_hba.conf

local   all             all                                    trust
host    all             all             127.0.0.1/32           md5
host    all             all             10.133.17.0/24         md5
host    replication     repuser         10.133.17.0/24         md5

service postgresql-9.3 start
psql -U postgres

create user ambari with password 'bigdata';
create user repuser with replication login password 'repuser';
CREATE DATABASE ambari owner ambari;
GRANT ALL PRIVILEGES ON DATABASE ambari to ambari;

在主/从节点安装完HControl Server后，执行
psql -U ambari ambari < /var/lib/ambari-server/resources/Ambari-DDL-Postgres-CREATE.sql

在主/从节点的/var/lib/pgsql/9.3/data目录下，各自建立recovery.done(主)和recovery.conf(从)文件，内容如下：

standby_mode=on
primary_conninfo='host=bdi_hcontrol_server user=repuser port=5432 password=repuser'
trigger_file='/tmp/trigger_file'
recovery_target_timeline = 'latest'

在主节点上运行：
rsync -ravztopg --exclude=recovery.done --exclude=pg_log /var/lib/pgsql/9.3/data/ root@bdi_B:/var/lib/pgsql/9.3/data/

重启主/从节点上的Postgresql 9.3
service postgresql-9.3 restart

3 安装配置HControl Server(主/从)

yum install ambari-server
ambari-server setup
在Enter advanced database configuration [y/n] (n)? 输入y
并选择[4] - PostgreSQL，其它设置成默认值即可。

4 安装配置HControl Agent

yum install ambari-agent
vim /etc/ambari-agent/conf/ambari-agent.ini

hostname=bdi_hcontrol_server

5 安装配置Keepalived

CentOS自带了Keepalived,直接 yum install keepalived即可。如果是其它linux，需要下载keepalived rpm包。

安装完毕后，主/从节点上修改/etc/keepalived/keepalived.conf配置文件如下：

! Configuration File for keepalived
global_defs {
  notification_email {
  alert@cmss.chinamobile.com
  }
  notification_email_from  alert@cmss.chinamobile.com
  smtp_server 127.0.0.1
  smtp_connect_timeout 30
  router_id HCONTROL_SERVER #此处可以设置为主机名
}
vrrp_script check_ambari_pg_status {
  script "/var/lib/ambari-server/resources/scripts/check_ambari_pg_status.sh"
  interval 3
}
vrrp_instance VI_1 {
  state BACKUP
  interface eth0 #此处根据实际情况调整
  virtual_router_id 51
  priority 50
  nopreempt  #减少不必要的主从切换
  advert_int 1
  authentication {
    auth_type PASS
    auth_pass 1111
  }
  virtual_ipaddress {
   10.133.17.241/24
  }
  notify_master "/var/lib/ambari-server/resources/scripts/to_master.sh"
  notify_backup "/var/lib/ambari-server/resources/scripts/to_backup.sh"
  notify_stop   "/var/lib/ambari-server/resources/scripts/to_stop.sh"
  track_script {
    check_ambari_pg_status
 }
}

当前配置下，先启动Keepalived的节点为Master节点。 启动命令为:/etc/init.d/keepalived start

check_ambari_pg_status.sh内容如下：

#!/bin/bash
count=$(ps -ef| grep -v grep | grep org.apache.ambari.server.controller.AmbariServer | wc -l)
if [ "${counter}" = "0" ]; then
   /etc/init.d/keepalived stop
   exit 1
fi

count=$(ps -ef| grep -v grep | grep postmaster | wc -l)
if [ "${counter}" = "0" ]; then
   /etc/init.d/keepalived stop
   exit 1
fi

to_master.sh内容如下：

#!/bin/bash
su -l postgres -c "touch /tmp/trigger_file"

to_backup.sh内容如下：

#!/bin/bash
su -l postgres -c "rm -rf /tmp/trigger_file"

to_stop.sh内容如下：

#!/bin/bash
service postgresql-9.3 stop
ambari-server stop

6 测试

在bdi_A和bdi_B上运行ip a命令，若发现节点绑定了虚拟IP(10.133.17.241),即为Master节点。 在Master节点运行如下命令：

psql -U postgres
select * from pg_stat_replication;

若发现一条client_addr为Slave节点IP的流复制的记录，则可认为流复制搭建成功。
在两边节点依次检查Keepalived、Postgresql、HControl Server进程，若都正常，则可认为HA 环境搭建成功。

7 故障恢复

假设场景：bdi_A为主服务器，bdi_B为从服务器，倘若bdi_A由于某种原因down掉导致bdi_B变为主服务器。在解除故障后，可以按如下步骤把bdi_A变为从服务器。
1.确保bdi_A上的keepalived,hcontrol server,postgresql9.3服务均关闭。
2.确保bdi_B上存在/var/lib/pgsql/9.3/data/recovery.done。如果不存在把recovery.conf变成recovery.done,内容见第2部分所描述。
3.在bdi_A上运行如下命令：

cd /var/lib/pgsql/9.3/data
rm -rf `ls | grep -v "pg_log"`
rm -rf pg_log/*
rsync -ravztopg  --exclude=postmaster.pid --exclude=pg_log  root@bdi_hcontrol_server:/var/lib/pgsql/9.3/data/  .
mv recovery.done recovery.conf
service postgresql-9.3 start
ambari-server start
service keepalived start

4.参考第6部分进行测试，若成功则表明成功把bdi_B变为从服务器。

附录 p8.4升级9.3方案

1.//停止ambari
ambari-server stop
2.//导出数据,密码是bigdata

5.service postgresql stop //停止pg8.4

把第一行的peer改成trust
vi /var/lib/pgsql/9.3/data/postgresql.conf
listen_addresses = '*'
7.//启动pg9.3
service postgresql-9.3 start
8./usr/pgsql-9.3/bin/psql -U postgres

//创建用户
create user ambari with password 'bigdata';
//创建数据库

//赋权限给ambari
GRANT ALL PRIVILEGES ON DATABASE ambari to ambari;
//退出
\q

9.//导入pg8.4数据
/usr/pgsql-9.3/bin/psql -U ambari ambari </usr/lib/hcontrol.bak
10.//重置ambari-server数据库
ambari-server stop
ambari-server setup
回车即可，但在Enter advanced database configuration [y/n] (n)?输入y，并选择4。
11.chkconfig postgresql off //关闭pg8.4自启动
12.ambari-server start
